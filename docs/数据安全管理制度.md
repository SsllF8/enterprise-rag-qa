# 星辰科技有限公司 数据安全管理制度

## 文档编号：SEC-2026-001
## 密级：内部公开
## 生效日期：2026年1月1日

---

## 一、总则

### 1.1 目的

为规范公司数据处理行为，保护公司及客户数据安全，防范数据泄露风险，根据《中华人民共和国数据安全法》《中华人民共和国个人信息保护法》及相关法律法规，制定本制度。

### 1.2 适用范围

本制度适用于公司全体员工、外包人员、实习生，以及所有涉及公司数据处理的场景。

### 1.3 数据分级分类

| 级别 | 定义 | 示例 | 访问权限 |
|------|------|------|----------|
| L1 公开 | 可对外公开的信息 | 公司官网内容、已发布的产品文档 | 所有人 |
| L2 内部 | 仅限公司内部使用 | 内部邮件、会议纪要、组织架构 | 全体员工 |
| L3 机密 | 仅限相关部门使用 | 客户合同、源代码、财务报表 | 部门内授权人员 |
| L4 绝密 | 极少数人可访问 | 核心算法、客户原始数据、安全密钥 | 指定人员，需审批 |

## 二、数据访问管理

### 2.1 账号权限管理

- 所有系统账号实行实名制，禁止共用账号
- 权限遵循「最小必要原则」，仅授予完成工作所需的最低权限
- 离职/转岗时，IT部门须在 24 小时内调整或回收相关权限
- 每季度进行一次权限审计，清理过期权限

### 2.2 数据访问申请

访问 L3 及以上级别数据需提交申请：
- L3 数据：部门经理审批
- L4 数据：部门总监 + 安全负责人双重审批
- 临时访问权限有效期最长 30 天，到期自动回收

### 2.3 数据脱敏

在开发和测试环境中使用数据时，必须进行脱敏处理：
- 个人信息：姓名、手机号、身份证号、邮箱
- 财务信息：银行卡号、交易金额
- 脱敏工具：公司统一提供的 DataMask 工具

脱敏规则示例：
- 手机号：138****5678
- 身份证号：110***********1234
- 银行卡号：6222 **** **** 1234

## 三、数据存储与传输

### 3.1 数据存储

- L2 及以上数据禁止存储在个人设备上
- 禁止使用个人云盘（百度网盘、iCloud 等）存储公司数据
- 开发环境不得存放生产数据
- 生产数据库定期备份，备份保留 90 天

### 3.2 数据传输

- 禁止通过个人微信、QQ 传输公司数据
- 外部传输必须使用公司邮箱或企业微信
- 传输 L3 及以上数据需加密（AES-256 或 SSL/TLS）
- 禁止将公司数据发送至个人邮箱

### 3.3 代码安全

- 禁止将公司代码推送到公开的 Git 仓库（GitHub、Gitee 公开仓库）
- 代码仓库访问权限与数据分级一致
- 提交代码前必须通过安全扫描（SonarQube + SAST）
- API Key、密码等凭证必须使用密钥管理系统，禁止硬编码

## 四、数据安全操作规范

### 4.1 笔记本电脑安全

- 开启全盘加密（BitLocker / FileVault）
- 设置开机密码，离开工位时锁定屏幕（Windows: Win+L）
- 禁止在公共 Wi-Fi 下处理公司数据，必须使用 VPN
- 每月更新一次系统补丁和安全软件

### 4.2 办公桌安全（桌面整洁原则）

- 离开工位时，纸质文件收入抽屉
- 禁止将包含敏感信息的纸质文件放在桌面过夜
- 含敏感信息的废弃文件须使用碎纸机销毁

### 4.3 会议安全

- 涉及 L3 及以上数据的会议，优先使用内部会议室
- 外部人员参会需提前报备，签署保密协议
- 会议室白板内容会后须擦除

## 五、事件响应

### 5.1 安全事件分类

| 级别 | 描述 | 响应时间 |
|------|------|----------|
| P0 紧急 | 大规模数据泄露、核心系统被入侵 | 15分钟内响应 |
| P1 严重 | 单个客户数据泄露、重要系统异常 | 1小时内响应 |
| P2 一般 | 安全策略违规、疑似泄露 | 4小时内响应 |
| P3 低级 | 安全意识不足、轻微违规 | 1个工作日内 |

### 5.2 应急流程

1. **发现**：任何员工发现安全事件，立即报告到 security@startech-ai.com
2. **评估**：安全团队 30 分钟内评估影响范围
3. **控制**：采取紧急措施止损（断网、停账号、回收权限）
4. **调查**：定位根因，保留证据
5. **修复**：消除漏洞，恢复服务
6. **复盘**：输出事件报告，修订安全策略

### 5.3 处罚

违反本制度的行为将按严重程度处理：
- 轻微违规：口头警告 + 安全培训
- 一般违规：书面警告 + 绩效扣分
- 严重违规：降级/降薪 + 通报批评
- 造成实际损失的：解除劳动合同，保留追究法律责任的权利

## 六、年度安全培训

- 全体员工：每年至少完成 1 次数据安全培训（线上）
- 新员工：入职第一周完成安全意识培训
- 技术岗位：每年至少完成 1 次安全编码培训
- 培训考核成绩纳入年度绩效参考
